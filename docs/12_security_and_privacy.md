# Security And Privacy

Evidence Gate is designed so organizational memory stays inside the partner's
chosen trust boundary by default.

## Default posture

The alpha in this repository does **not** send repository content, Jira issues,
Confluence pages, Slack threads, or PagerDuty incidents to a public LLM API by
default.

Current defaults:

- embeddings backend: `local_structural`
- decision backend: `deterministic`
- remote inference allowed: `false`
- public model training allowed: `false`
- organizational memory sandboxed: `true`

That means the checked-in structural encoder, retrieval stack, and decision
engine operate on local files and local persisted state under
`EVIDENCE_GATE_AUDIT_ROOT`, `EVIDENCE_GATE_KB_ROOT`, and `EVIDENCE_GATE_AST_CACHE_ROOT`.

## Organizational memory boundaries

Evidence Gate stores and indexes:

- repository code and tests
- internal docs and runbooks
- Jira, Confluence, Slack, GitHub, and PagerDuty context that you explicitly
  ingest or sync
- audit decisions generated by the gate

Those assets are not used for public model training by the application itself.
If you configure a remote inference backend, only that configured backend sees
the prompt payloads you choose to send there. Evidence Gate does not
independently forward indexed corpus data to any public training pipeline.

## Secure backend configuration

Set the backend contract through environment variables.

### Local-only default

```bash
export EVIDENCE_GATE_EMBEDDING_BACKEND=local_structural
export EVIDENCE_GATE_DECISION_BACKEND=deterministic
export EVIDENCE_GATE_REMOTE_INFERENCE_ALLOWED=false
export EVIDENCE_GATE_PUBLIC_MODEL_TRAINING_ALLOWED=false
```

### Azure OpenAI

Use this when your organization already has an approved Azure OpenAI tenancy.

```bash
export EVIDENCE_GATE_REMOTE_INFERENCE_ALLOWED=true
export EVIDENCE_GATE_EMBEDDING_BACKEND=azure_openai
export EVIDENCE_GATE_EMBEDDING_ENDPOINT=https://example.openai.azure.com
export EVIDENCE_GATE_EMBEDDING_DEPLOYMENT=text-embedding-3-large
export EVIDENCE_GATE_EMBEDDING_API_VERSION=2024-10-21

export EVIDENCE_GATE_DECISION_BACKEND=azure_openai
export EVIDENCE_GATE_DECISION_ENDPOINT=https://example.openai.azure.com
export EVIDENCE_GATE_DECISION_DEPLOYMENT=gpt-4.1
export EVIDENCE_GATE_DECISION_API_VERSION=2024-10-21
```

### Ollama

Use this when the enterprise requires local inference on a workstation or
inside a private network.

```bash
export EVIDENCE_GATE_REMOTE_INFERENCE_ALLOWED=true
export EVIDENCE_GATE_EMBEDDING_BACKEND=ollama
export EVIDENCE_GATE_EMBEDDING_ENDPOINT=http://127.0.0.1:11434
export EVIDENCE_GATE_EMBEDDING_MODEL=nomic-embed-text

export EVIDENCE_GATE_DECISION_BACKEND=ollama
export EVIDENCE_GATE_DECISION_ENDPOINT=http://127.0.0.1:11434
export EVIDENCE_GATE_DECISION_MODEL=llama3.1:8b
```

### vLLM

Use this when the enterprise runs a self-hosted OpenAI-compatible inference
gateway.

```bash
export EVIDENCE_GATE_REMOTE_INFERENCE_ALLOWED=true
export EVIDENCE_GATE_EMBEDDING_BACKEND=vllm
export EVIDENCE_GATE_EMBEDDING_ENDPOINT=http://vllm.internal:8000/v1
export EVIDENCE_GATE_EMBEDDING_MODEL=text-embedding-model

export EVIDENCE_GATE_DECISION_BACKEND=vllm
export EVIDENCE_GATE_DECISION_ENDPOINT=http://vllm.internal:8000/v1
export EVIDENCE_GATE_DECISION_MODEL=instruction-model
```

## Storage and sandboxing

Recommended practice:

- keep audit, knowledge-base, and AST cache roots outside the checked-out repo
- mount live connector exports into the same private filesystem namespace as the
  API service
- use read-only API tokens for Jira, Confluence, Slack, GitHub, and PagerDuty
- prefer `GATING_MODE=shadow` during the first pilot so no production merge is
  blocked before the partner reviews the evidence

Example:

```bash
export EVIDENCE_GATE_AUDIT_ROOT=/srv/evidence-gate/audit
export EVIDENCE_GATE_KB_ROOT=/srv/evidence-gate/knowledge_bases
export EVIDENCE_GATE_AST_CACHE_ROOT=/srv/evidence-gate/ast_graphs
```

## What this document does not claim

- It is not a legal promise about a third-party model vendor's retention policy.
- It does not mean every future backend adapter is implemented in this alpha.
- It does mean the application now has an explicit configuration path for
  Azure OpenAI, Ollama, and vLLM, and the safe default remains local-only.
